Download Engine — Segment + Multi-Thread Architecture

Core idea: A 1 GB file is split into 16 segments of ~64 MB each. Each segment is downloaded by 2–4 threads pulling sequential sub-ranges within that segment. All 16 segments download in parallel. When all sub-ranges of a segment complete, that segment is marked done. When all segments complete, BDM merges them into the final file via zero-copy assembly. This is faster than both single-thread and single-thread-per-segment approaches.

Per-segment threading: Each segment is further split into 2–4 sub-ranges. Thread count is auto-tuned: 2 threads for segments <50 MB, 4 threads for >200 MB. Each thread uses its own URLSession data task with a Range header for its sub-range. All threads write to the same pre-allocated sparse file using pwrite() at their exact byte offset — no temp files, no copy, no merge step needed.
Assembly strategy: Since all threads write directly to correct offsets in a single sparse file, "assembly" is just: (1) verify all byte ranges are complete, (2) optional SHA-256 verify, (3) atomic rename from .bdm-partial to final filename. Zero-copy. The whole operation is a single rename() syscall.
Adaptive speed: Every 500ms, the engine measures throughput per thread. If a thread is bottlenecked (slow CDN shard, congested route), it is cancelled and its remaining sub-range is reassigned to a new connection. Segments that finish early donate their thread capacity to the slowest active segment.

JSON Localization System

No .strings, no .stringsdict. BDM uses flat JSON files for localization. One file per language. Community translators fork the repo, add a {lang}.json, and submit a PR. The app ships with en.json and loads others from ~/Library/Application Support/BDM/Locales/ at runtime.
en.json (bundled default)
{ "app.name": "BlackSwan Download Manager", "menu.add_urls": "Add URLs", "menu.pause_all": "Pause All", "menu.resume_all": "Resume All", "filter.all": "All", "filter.active": "Active", "filter.done": "Done", "filter.failed": "Failed", "notif.complete": "{{file}} — Complete", "notif.complete.body": "{{size}} in {{time}}", "detail.segments": "{{n}} segments × {{t}} threads", "settings.export": "Export Settings", "settings.import": "Import Settings", "view.detailed": "Detailed", "view.compact": "Compact", "view.minimal": "Minimal" }

How it works: (1) BDMLocalizer class loads en.json from app bundle at launch. (2) If user has selected another language in settings, overlay that JSON on top — missing keys fall back to English. (3) {{var}} placeholders resolved at runtime via string interpolation. (4) RTL languages auto-detected from JSON metadata key "_direction": "rtl".
For translators: Fork repo → copy en.json → rename to your lang code → translate values (never keys) → PR. BDM validates JSON on load and reports any missing keys to Console.app. Partial translations are fine — untranslated keys show English.

Additional Features

Engine

Intelligent segment sizing

Smaller segments for slow servers (more parallelism), larger segments for fast CDNs (less overhead). Learned per-domain over time.

Engine

Mirror / fallback URLs

Add multiple mirrors for the same file. If primary stalls, remaining segments pull from the mirror. Common for Linux ISOs.

Engine

Speed test on first connect

Brief 2-second burst test on first segment → auto-set optimal segment count and threads. No user tuning needed.

UX

Download notes / comments

Right-click → Add Note on any download. "This is for the staging server" or "Re-download after v2.1 ships". Searchable.

UX

Download history with stats

Total downloaded this week / month / all time. Average speed. Top domains. Exportable. Fun and useful for tracking bandwidth usage.

UX

Global hotkey to show/hide

Configurable system-wide hotkey (e.g. ⌥⌘D) toggles the BDM window. Like Spotlight for downloads.

UX

Drag files out of BDM

Drag a completed download row directly into Finder, Mail, Slack, etc. Standard NSPasteboard file promise.

Network

Network interface selector

Bind downloads to Wi-Fi, Ethernet, or VPN. Large downloads on Ethernet, small ones on whatever.

Network

Auto-pause on metered Wi-Fi

Detect personal hotspot or known metered networks. Auto-pause queue. Resume when back on home network.

Automation

Watched folder integration

Point BDM at a folder. Drop a .url or .txt file in → auto-import and queue the URLs. CI/CD friendly.

Automation

Webhooks on events

POST to a URL when download completes/fails. JSON payload with filename, size, path, duration. Integrate with Slack, Discord, Home Assistant.

Automation

AppleScript / JXA dictionary

Full scriptable app with AppleScript dictionary. tell app "BDM" to add url "..." with segments 16

Security

TLS cert inspector

Detail panel shows cert chain, issuer, expiry. Warns on self-signed or expired. Never silently accepts bad certs.

Security

Quarantine flag management

Show Gatekeeper status per file. Option to clear quarantine flag from detail panel (with permission prompt).

Security

Malware scan integration

Pass completed downloads to XProtect / custom antivirus via NSWorkspace. Show scan result in detail panel.

i18n

In-app translation editor

Power users can edit translations directly in Settings → Language → Edit. Changes saved to user's locale .json. Submit upstream via "Contribute Translation" button.

i18n

Pluralization rules in JSON

"items_count" key supports {"one": "1 file", "other": "{{n}} files"} nested objects. CLDR plural rules per locale.

UX

Widget for macOS Desktop

WidgetKit widget showing active downloads with progress. Small, medium, large sizes. Interactive in macOS Sonoma+.

Build Roadmap

Phase 1 · Engine Core

Multi-Thread Segment Downloader

The heart of BDM. If this isn't fast, nothing else matters.

  • XPC download service: com.amirhpcom.bdm.downloader
  • SegmentPlanner: HEAD check → split into N segments
  • ThreadedSegment actor: 2–4 sub-range tasks per segment via TaskGroup
  • Sparse file writer: pre-allocate → pwrite() at byte offset per thread
  • Assembly: verify all ranges complete → SHA-256 → atomic rename()
  • Adaptive rebalancing: 500ms throughput measurement, slow-thread reassign
  • Resume: persist per-thread byte offsets to SwiftData on pause/crash
  • Token bucket bandwidth limiter (global + per-download)
  • Minimal SwiftUI window to test: add URL → watch segments fill
Phase 2 · UI + View Modes

Three Views, Filtering, Sidebar

  • Detailed view with segment map, path, speed, ETA
  • Compact view: single-line with progress bar + percent

  • View toggle in titlebar (⌘1/⌘2/⌘3)
  • Filter bar: All / Active / Paused / Done / Failed with count badges
  • Collapsible sidebar (⌘⌥S, icon-only mode)
  • Search (⌘F) + Sort (name, size, speed, date, status)
  • Right-click context menu: Copy URL, Copy Path, Reveal, Quick Look, Re-download, Delete
